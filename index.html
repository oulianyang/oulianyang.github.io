hexo_本地图片引用问题

发表于 2019-12-01

hexo4.0图片引用

步骤

  1. 在根目录下配置文件_config.yml 中 post_asset_folder:false改为true

  2. npm install https://github.com/7ym0n/hexo-asset-image –save

  3. 执行以上两步后在新建 hexo n xxxx博客名 就会同时在当前目录生成一个 同名文件夹，将需要的图片放入其中

  4. 本地测试需要重启本地服务,清空缓存

hexo_导航图标异常

发表于 2019-12-01

hexo 导航配置

修改对应主题下的_config.yml文件

menu:
  home: / || home
  archives: /archives/ || archive

# Enable/Disable menu icons.
menu_icons:
  enable: true

上面是初始配置，发现导航不能正常，查询说是修改如下：

menu:
  home: /
  archives: /archives/
menu_icons:
  enable: true
  home: home
  archives: home

导航生效了但是导航字体图标丢失了，下面的menu_icons貌似没有作用，于是查找问题，查找打包源码如下：

｛% for name, path in theme.menu %｝
    {% set itemName = name.toLowerCase() %}
    <li class="menu-item menu-item-{{ itemName | replace(' ', '-') }}">
      <a href="{{ url_for(path.split('||')[0]) | trim }}" rel="section">
        {% if theme.menu_icons.enable %}
               

            {% endif %}
        {{ __('menu.' + name) | replace('menu.', '') }}
      </a>
    </li>
  {% endfor %}

源码并没有取menu_icons的配置，直接取的menu的配置以||分隔，所以初始配置应该是正确的

menu:
  home: / || home
  archives: /archives/ || archive

# Enable/Disable menu icons.
menu_icons:
  enable: true

这样恢复配置后导航失效，看了下链接，多了空格，于是去掉默认配置多余的空格：

menu:
home: /||home
archives: /archives/||archive

# Enable/Disable menu icons.
menu_icons:
  enable: true

好这下正常了
